## Tipo de Mudança
- [ ] 🐛 Correção de Bug (mudança não-quebrável que corrige um problema)
- [ ] ✨ Nova Funcionalidade (mudança não-quebrável que adiciona algo novo)
- [ ] 💥 Breaking Change (correção ou funcionalidade que causa mudança em comportamentos existentes)
- [ ] ♻️ Refatoração (melhoria de código que não altera o comportamento)
- [ ] 📚 Documentação (atualizações em READMEs, diagramas, etc.)
- [ ] ⚙️ Infra/Configuração (ex: Dockerfile, CI/CD)

## Checklist de Qualidade
- [ ] Meu código segue os padrões de projeto estabelecidos.
- [ ] Realizei uma auto-revisão do meu próprio código.
- [ ] Comentei meu código de forma clara, especialmente em lógicas complexas.
- [ ] Atualizei a documentação e os diagramas (se necessário).
- [ ] As alterações não geram novos erros ou *warnings* no console/build.
